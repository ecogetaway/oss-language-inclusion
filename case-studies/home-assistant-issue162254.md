# Case Study: home-assistant/core — Translation String Fix — Issue #162254

## Repository
[home-assistant/core](https://github.com/home-assistant/core)

## Contribution Type
Issue / Translation Fix

## Language
Multiple locales (translation string fix)

## What Happened
A contributor submitted a translation string fix to Home Assistant — one of the most actively maintained open-source projects on GitHub with over 70,000 stars. The contributor asked repeatedly for help with labelling and CLA (Contributor License Agreement) requirements — standard administrative steps that are straightforward for code contributors but poorly documented for language contributors. No maintainer responded to guide them through the process. The issue was stale-closed after approximately 82 days.

## Timeline
- Opened: 2025-2026
- Closed: ~82 days later
- Days open: ~82

## Closing Reason
Bot-stale — contributor actively seeking help, no maintainer response

## Maintainer Response
None — contributor's requests for CLA and labelling help went unanswered

## Why This Matters
Home Assistant is one of the most well-resourced and actively maintained open-source projects globally. If a translation contributor cannot get basic administrative help on a project of this scale, the structural gap is systemic — not a resource problem specific to smaller projects. The contributor was willing, engaged, and persistent. The infrastructure to support them simply did not exist.
