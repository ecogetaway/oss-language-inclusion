# Maintainer Feedback

This folder is the **in-repo landing place** for maintainer experience; the usual entry point is a **GitHub issue** (not a file here).

This folder contains **raw maintainer input** (issues, short notes, linked artifacts).
Processed patterns are maintained in [`signals/maintainer-signals.md`](../signals/maintainer-signals.md).

**Start here:** [`CONTRIBUTING.md`](../CONTRIBUTING.md) and **[New issue (choose a template)](https://github.com/ecogetaway/oss-language-inclusion/issues/new/choose)**.

## How to contribute

Open an issue (template **Maintainer feedback**) or a small PR with a short note. Public links are preferred for verification.

Use this format:

```text
Project:
Role:
Challenge:
What would help:
```
